HyperText Markup Language

Basics of the page. The text and skeleton. Nothing fancy here. No alignments, no styling, no color, just letters at different sizes.

In fact, the only thing I would really note are classes, and ids. It marks them in special ways.

Cascading Style Sheets (CSS)

CSS is where much of the magic happens. You can take that text (markup) and change its appearance. The key thing that I noted is to never forget the cascading. If you get out of order with the different styles, you end up doing way more coding and the functioning seems less precise.

Truthfully, I still do a fair share of commenting out styles and then seeing if it is even necessary to narrow things down.

Bootstrap

Bootstrap is a CSS Library. Those different styles you made in CSS? Bootstrap has a plethora of premade ones. It makes mobile responsiveness and navigation bars, cards, and other components an absolute breeze.

It has "limitations" like any other piece of the puzzle. I have found that styling them to my own liking is kind of obnoxious if you become too attached to the styling premades.

Even with my focus on the vanilla languages (HTML, CSS, js), I find myself wanting to launch bootstrap, copy the parameters I like, and then move forward.

JavaScript

Other than day 1, where I was crashing my browser using while loops, it has been going very well. I have a couple projects uploaded, and I understand how it works. It is just a matter of knowing what I want to do with it. Without knowing any storage methods except localStorage (I've been really focusing on vanilla JavaScript), it is difficult to apply anything real interesting.

Using this to add styling and elements is an infinitely valuable tool.
