HTML

I picked this up quite quick. It is a matter of organization and documentation. Having taken multiple courses in accounting, I was well prepared to handle this. Basically, everything gets stacked around eachother.

In it's most basic form, you will have a main div around the entire content. Within that, there are sub-div for narrowing the page down further. You eventually get to more specific markup. But overall, it is a very simple concept.

You can stack names on top of the different elements, and link them to CSS. This is similar to the accounts you are crediting and debiting in bookkeeping.

CSS

Css is where a lot of the magic happens. You can take that markup and change its style attributes and even add animations. This one was a bit more difficult. The overall structure, layout, and properties is easy enough. Making sure it is DRY is a bit annoying right now.

You just keep narrowing it down, making sure you are using shortcuts as much as possible, trying to learn remember the different shortcuts, etc until you get it looking good.

Bootstrap

Ah, bootstrap, I love you. Like I said earlier, I dive into the deep end. After about 2 hours of html and css combined, I jumped into the library and started putting pieces together. I realized I should go back to learn HTML and CSS fundamentals. So that's what I did.

Now that I have those fundamentals (make no mistake, it's not perfect understanding), Bootstrap is even more fun and useful. I've also done a very little bit with Materialize.css and skeleton.css, but Bootstrap is my favorite and strongest right now.

Once I get more practice with JavaScript, I will be looking at Materialize more closely.

JavaScript

Other than day 1, where I was crashing my browser using while loops, it has been going very well. I have a couple projects uploaded, and I understand how it works. It is just a matter of knowing what I want to do with it. Without knowing any storage methods except localStorage (I've been really focusing on vanilla JavaScript), it is difficult to apply anything real interesting.

Using this to add styling and elements is an infinitely valuee tool.
